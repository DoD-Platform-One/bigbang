# Package Lifecycle: Onboarding

Use this process-oriented checklist to prepare a package for Big Bang integration.

- [ ] 1. [Get BBTOC Approval](https://repo1.dso.mil/big-bang/product/bbtoc/-/blob/master/process/Package%20Maintenance%20Tracks.md): Align the package with the appropriate maintenance track and approval flow.
- [ ] 2. Notify Cyber: Notify the Cyber team of the package onboarding effort so security review can start early.
- [ ] 3. Confirm ownership model: Review [Package Owner Requirements & Overview](../package-integration/ownership.md) and make sure owner expectations are covered.
- [ ] 4. Prepare development baseline: Complete package development tasks in [Develop a Big Bang Package](../develop-package.md).
- [ ] 5. Start technical integration: Continue with [Package Lifecycle: Integration](integration.md).

After the technical integration work is complete and validated, proceed with a Big Bang MR and normal review/release workflows.
